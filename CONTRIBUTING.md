# Contributing to Azure Arc Software Assurance Dashboard

Thank you for your interest in contributing to this project! We welcome contributions from the community.

## How to Contribute

### Reporting Issues
- Use the GitHub issue tracker to report bugs or request features
- Provide detailed information including steps to reproduce
- Include screenshots or workbook exports when helpful

### Submitting Changes
1. Fork the repository
2. Create a new branch for your feature or fix
3. Make your changes
4. Test your workbook modifications
5. Submit a pull request with a clear description

### Code Standards
- Follow Azure Resource Graph query best practices
- Use clear, descriptive names for workbook elements
- Include comments in complex queries
- Test with multiple subscription scenarios

### Workbook Development Guidelines
- Maintain backward compatibility when possible
- Use parameterization for flexibility
- Follow Azure Monitor Workbooks design patterns
- Optimize queries for performance

## Getting Help
- Review existing issues and discussions
- Check Azure Arc and Azure Monitor documentation
- Reach out through GitHub issues for support

Thank you for contributing!
